# BrainstormBuddy — гид по главному экрану

> Обновлено: 2026-07-04, соответствует сборке ветки `feature/chatlog-restore`.
> Это пользовательская документация оверлея: все кнопки, контекст-меню, эквалайзер.
> Для агентов: описания соответствуют обработчикам в `OverlayWindow.xaml(.cs)`.

## 1. Тайтлбар

Перетаскивание за любое место полосы = переместить окно. **Двойной клик = сквозной режим** (см. «Глаз»).

| Элемент | Что делает |
|---|---|
| Селектор промпта | Мгновенно меняет роль LLM (Собеседование / Совещание / …). ВНИМАНИЕ: сбрасывает весь диалог — историю LLM и счётчик токенов (новый промпт со старым контекстом даёт кашу, так задумано) |
| Чип **REC** / Чип **LLM** | Индикаторы, не кнопки: пульс красным = идёт запись, пульс синим = ждём ответ LLM. Оба серые = пауза |
| Робот | Пауза LLM (`Ctrl+Shift+Y`): аудио и распознавание продолжаются, ответы не запрашиваются. Удобно на NDA-темах |
| Пауза ‖ | Пауза аудио (`Ctrl+Shift+P`): полная остановка захвата звука |
| Шестерёнка | Настройки (`Ctrl+Shift+S`) |
| Автоскролл ↕ | Вкл/выкл автопрокрутку истории к новым ответам (выключен — иконка тускнеет) |

## 2. Тулбар

| Элемент | Что делает |
|---|---|
| Слайдер **ТИШИНА** (0.4–10 с) | Пауза в речи, после которой реплика уходит в распознавание |
| Слайдер **ЧАНК МАКС** (8–120 с) | Максимальная длина куска аудио — страховка на непрерывную речь |
| Молния (жёлтая) | Отправить сейчас: не ждать паузу, немедленно отдать накопленное аудио в STT |
| Экспорт | Сохранить весь транскрипт беседы в файл |
| AI-протокол | LLM составляет протокол встречи: тезисы, решения, action items |
| Камера | Режим скриншота (`Ctrl+Shift+C`): временно показать окно в захвате экрана — иначе оно невидимо в OBS/Zoom и на скриншотах |
| « · ↺ · » | Пресеты темпа речи: медленнее (пауза длиннее) / сброс к 1.8с·60с / быстрее |
| Глаз | Сквозной режим: клики проходят «сквозь» окно в приложение под ним. **Выход — только двойной клик по тайтлбару** (единственная «опасная» кнопка) |
| Корзина (красная) | Очистить историю и начать новый чат: сбрасывает контекст LLM и счётчик токенов |
| Крест (красный) | Скрыть оверлей (вернуть: `Ctrl+Shift+H`). Приложение продолжает слушать |

## 3. История вопрос-ответ

Каждая пара: **ВОПРОС** (распознанная речь, mono-тайминги `←STT · длительность`) и карточка
**ОТВЕТ** с суммарной задержкой Σ (зелёная <1.5 с / жёлтая / красная >3.5 с).

Точка слева у пары: синяя = отправляется, зелёная = готово, красная = ошибка.

Дополнительные блоки при мульти-агентах:
- синий **HRD СОВЕТ** — совет HR-агента (появляется только на soft-skills темах, иначе агент молчит);
- янтарный **ЗАЩИТА** — реакция на давление/манипуляцию в разговоре;
- золотой **РАСШИРЕННЫЙ ОТВЕТ** — появляется после «Ответить расширенно».

### Правый клик по паре — контекст-меню

| Пункт | Что произойдёт |
|---|---|
| Копировать всё | Вопрос + ответ в буфер обмена |
| Копировать вопрос | Только распознанный текст |
| Копировать ответ | Только ответ LLM |
| Ответить расширенно | LLM отвечает подробно — под парой появляется золотой блок |
| Сбросить расширенный ответ | Убирает золотой блок |
| Фокус на этой паре | Зелёная рамка — пометить важное (фокус только у одной пары одновременно) |
| Снять фокус со всех | Убрать рамку |

## 4. Эквалайзер (лента АУДИО)

Лента последних 2 минут звука: один бар = полсекунды, новое появляется справа, старое
уезжает влево. Клик по заголовку **▾ АУДИО** сворачивает ленту (шеврон меняется на ▸),
состояние запоминается между перезапусками. Под лентой — шкала абсолютного времени
(HH:mm:ss каждые 10 секунд), фазы сопоставляются с mono-таймингами в карточках истории.

### Цвета

| Цвет | Значение |
|---|---|
| Розовый `#F87171` | Идёт живая речь; высота бара = громкость в этот момент |
| Синий высокий `#60A5FA` | Реплика вырезана и отправлена в распознавание — STT работает |
| Зелёный высокий `#4ADE80` | Ответ LLM получен и показан в истории |
| Тускло-бордовый `#9C5A5A` | Звук есть, но система в простое (фон/музыка между репликами) |
| Серые пеньки `#3A4653` | Тишина ниже порога VAD; при паузе лента тоже плоская |

Цветовая логика едина по всему окну (точки статусов, чипы, эквалайзер):
**розовый/красный = вход, синий = обработка, зелёный = результат.**

### Что это даёт: диагностика одним взглядом

| Вопрос | Как ответить по ленте |
|---|---|
| Звук вообще пишется? | Люди говорят, а лента плоская → проблема с микрофоном/лупбэком |
| Реплика дошла до конца? | После розового должен появиться синий, потом зелёный. Розовый есть, синего нет → VAD не режет (речь без пауз — жми молнию). Синий есть, зелёного нет → упал STT или LLM |
| Какая задержка? | Расстояние розовый→зелёный на шкале времени = сколько секунд ждать подсказку |
| Порог VAD верный? | Тихая речь красится тускло-бордовым, а не розовым → порог RMS завышен, крутить в Настройки→Аудио |
| Пауза не забыта? | Лента внезапно плоская при живом разговоре → ты на паузе (чипы REC/LLM серые) |

## 5. Футер

| Элемент | Что делает |
|---|---|
| Токены: N (in/out) | Суммарный расход токенов LLM за сессию, включая всех агентов |
| Часы | Текущее локальное время (mono) — для сверки с таймингами |
| Поле «Секретное сообщение в LLM…» | Ручной вопрос LLM с меткой `[secret]` — обрабатывается приоритетно. Отправка: Enter или круглая кнопка |

## 6. Глобальные горячие клавиши

Работают, даже когда окно скрыто или не в фокусе:

| Комбинация | Действие |
|---|---|
| `Ctrl+Shift+H` | Показать/скрыть оверлей |
| `Ctrl+Shift+P` | Пауза аудио |
| `Ctrl+Shift+Y` | Пауза LLM |
| `Ctrl+Shift+S` | Настройки |
| `Ctrl+Shift+L` | Окно логов |
| `Ctrl+Shift+C` | Режим скриншота (показать окно в захвате экрана) |
| `Ctrl+Shift+O` | Прозрачность окна |
