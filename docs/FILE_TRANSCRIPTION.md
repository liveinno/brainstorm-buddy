# Транскрибация локальных файлов

Отдельное окно: распознавание готового аудио/видео-файла встроенным движком STT
(GigaAM ONNX) — **локально, без интернета** — с тайм-кодами, AI-саммари, историей
и экспортом.

## Как открыть
- **Настройки → Локальный STT → «Открыть транскрибацию файла →»**.
- Или запуск с ключом `--file-transcription` (deep-link, открывает окно на старте).

## Что умеет
- Выбрать файл (mp4, webm, mkv, mov, avi, m4a, mp3, wav, aac, ogg, opus, flac).
- **Транскрипт с тайм-кодами** `[mm:ss] …` (посегментно).
- **AI-саммари** транскрипта через текущий LLM (по умолчанию локальный Ollama).
- Копирование и «Скачать .txt» отдельно для транскрипта и для саммари; в текстовых
  полях работает выделение и ПКМ → «Выделить всё / Копировать».
- **История** транскрибаций слева (как чат): клик — открыть прошлую; ПКМ — удалить.
- Красивый прогресс-бар (индетерминантный «бегущий» блик на извлечении аудио,
  затем определённый прогресс на распознавании).

## Как устроено (для разработчика)
Поток: **извлечение аудио → сегментация с тайм-кодами → GigaAM → (опц.) саммари**.

| Слой | Файл |
|---|---|
| Извлечение аудио 16кГц mono | `BrainstormBuddy.Core/Media/MediaAudioExtractor.cs` |
| Транскрибация с тайм-кодами | `BrainstormBuddy.Core/Stt/FileTranscriptionService.cs` |
| История (JSON на запись) | `BrainstormBuddy.Core/Transcription/TranscriptionHistoryStore.cs` |
| Окно (UI) | `BrainstormBuddy/FileTranscriptionWindow.xaml(.cs)` |
| Точка входа + движок | `App.ShowFileTranscription()` / `App.ResolveNativeSttEngine(...)` |

- **Аудио.** mp4/m4a/mp3/wav/wma → Windows **Media Foundation** (`MediaFoundationReader`,
  без внешних зависимостей). webm/opus/ogg/mkv → **ffmpeg** (у Media Foundation обычно
  нет этих кодеков) — см. раздел «ffmpeg» ниже.
- **Тайм-коды.** Поток 16кГц режется на окна ~18с (меньше внутреннего чанка движка
  24с — идут одним проходом), граница подвигается к тихому месту, чтобы реже резать
  слова. Тайм-код = смещение начала окна. Пословных меток нет (CTC-кадр = 40мс — можно
  добавить позже правкой `GigaamCtcDecoder`).
- **Движок.** `App.ResolveNativeSttEngine` переиспользует уже загруженный `App.SttEngine`,
  если он native (без повторной загрузки модели ~900МБ). `MelFrontend` и ONNX-сессия
  потокобезопасны → совместная работа с «живым» STT безопасна.
- **Саммари.** `IApiClient.AskAsync` (транскрипт в system-prompt). Длинный текст —
  map-reduce: части → частичные конспекты → финальное сведение (обходит лимит контекста).
- **История.** `%APPDATA%\BrainstormBuddy\transcriptions\*.json`.

## Движок Whisper (turbo) — качество

В окне транскрибации есть выбор движка:
- **GigaAM** — быстрый, русский, без пунктуации (дефолт; он же для живого суфлёра).
- **Whisper turbo** — пунктуация, заглавные, английский/смешанная речь, **нативные
  тайм-коды сегментов**. Тяжелее (RTF ~1–1.5 на CPU против ~0.05 у GigaAM), поэтому
  только для готовых файлов, не для живого суфлёра.

Реализация: `WhisperSttEngine` (Core/Stt) через **Whisper.net** (whisper.cpp), нативно,
без Docker/Python. Модель `ggml-large-v3-turbo-q5_0.bin` (~547 МБ) качается отдельно:
**Настройки → Локальный STT → «Скачать модель Whisper»** (в `%APPDATA%\...\models`).
Движок кэшируется в `App` (модель грузится один раз). Оба движка отдают
`IFileTranscriber` → окно работает с любым единообразно.

- **GPU-ускорение (Vulkan).** Пакет `Whisper.net.Runtime.Vulkan` — работает на
  NVIDIA/AMD/Intel через драйвер (без CUDA). Настройка «Ускорение Whisper»: **Авто**
  (дискретная видеокарта → GPU; иначе CPU, т.к. встройка через Vulkan обычно медленнее
  CPU) или **CPU**. На ноутбуках Vulkan-устройство 0 — встройка, поэтому целимся в
  дискретку через `GGML_VK_VISIBLE_DEVICES` (индекс из `GpuEnumerator.BestDiscreteIndex`).
  Порядок рантаймов — `RuntimeOptions.RuntimeLibraryOrder` (до первой загрузки либы).
  Замер на RTX 4050: 20-сек клип **14с на GPU против 31с на CPU** (на длинных файлах
  выигрыш больше — загрузка модели амортизируется). Тест: `--whisper-file <media> <model> gpu <vkDevice>`.
- **Проверка оборудования** (Настройки → Локальный STT → «Проверить оборудование»):
  по CPU/ОЗУ/GPU даёт вердикт 🟢/🟡/🔴 и рекомендует движок.
- Whisper требует 16 кГц mono — подаём уже готовые сэмплы из `MediaAudioExtractor`.
- Проверка ядра: `Headless --whisper-file <media> [ggml-model]` — печатает транскрипт
  с пунктуацией и тайм-кодами. На 22-сек образце дал 7 сегментов с пунктуацией.

## ffmpeg (для webm / mkv / opus)
mp4 / mp3 / m4a / wav работают без ffmpeg (Media Foundation). Для webm/mkv/opus нужен
ffmpeg — три уровня, по приоритету поиска `MediaAudioExtractor.TryFindFfmpeg`:
1. **Бандл из инсталлятора** — `{app}\ffmpeg\ffmpeg.exe`. В мастере установки есть
   **галочка «ffmpeg»** (компонент включается, если положить `ffmpeg.exe` в
   `packaging\inno\ffmpeg\` перед сборкой — см. `#define HaveFfmpeg` в `.iss`).
2. **Докачка в приложении** — `%APPDATA%\BrainstormBuddy\ffmpeg\ffmpeg.exe`. Если ffmpeg
   не найден и выбран webm, в окне появляется жёлтая плашка с кнопкой **«Скачать ffmpeg
   (~80 МБ)»** → `FfmpegInstaller` тянет официальную сборку (gyan.dev, рекомендована
   на ffmpeg.org) и распаковывает `ffmpeg.exe`. Нужен интернет один раз.
3. **Системный `PATH`** — если ffmpeg уже установлен глобально.

Нет интернета и нет бандла → понятное сообщение с путём, куда положить `ffmpeg.exe`
вручную. Лицензия ffmpeg — GPL/LGPL, распространяется отдельно (ffmpeg.org).

Проверка загрузчика: `BrainstormBuddy.Headless.exe --install-ffmpeg <dir>` (скачивает,
распаковывает, печатает версию).

## Проверка (CLI)
Ядро пайплайна гоняется без UI:
```
BrainstormBuddy.Headless.exe --transcribe-file <media> [onnx] [labels]
```
Проверено на mp4 (Media Foundation) и webm (ffmpeg) из 20-сек. образца речи —
корректный русский транскрипт с тайм-кодами.
