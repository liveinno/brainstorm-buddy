# Интерактивное обучение (coach marks)

Первый запуск (и «Настройки → Показать обучение заново») запускает **интерактивный
тур**: полупрозрачный слой поверх окна настроек **подсвечивает реальные контролы
рамкой** и показывает выноску с пояснением (Далее / Назад / Пропустить). Заменил
прежнее слайд-шоу (`WizardWindow` удалён).

**Строго без затемнения экрана.** Фон слоя почти прозрачный (`#01000000`) — только
рамка-подсветка + карточка. Никакого чёрного фуллскрина (была проблема с блокировкой
экрана — не повторяем).

## Файлы
- `BrainstormBuddy/CoachMarkOverlay.xaml(.cs)` — слой: рамка-подсветка + выноска,
  «приклеен» к целевому окну (следует за перемещением/ресайзом), Esc = пропустить.
- `App.ShowOnboarding()` — открывает настройки и запускает тур; шаги в
  `App.BuildCoachSteps()` (движок STT → CPU/GPU → выбор видеокарты → транскрибация
  файлов → Сохранить). Deep-link `--coach`.
- `CoachStep` — модель шага: `Title`, `Body`, `Target` (какой контрол подсветить),
  `Prepare` (подготовка перед подсветкой).

## Как это работает
- **Позиционирование.** Экранный прямоугольник контрола берётся через
  `PointToScreen`, переводится в координаты слоя через `PointFromScreen` — это
  корректно учитывает DPI. Слой накрывает целевое окно 1:1 (та же система координат).
  Проверено: подсветка совпала с реальным `SttEngineCombo` попиксельно.
- **Грабли TabControl.** Контент невыбранной вкладки `TabControl` ОТСУТСТВУЕТ в
  визуальном дереве, пока вкладку не выбрали. Поэтому поиск родительского `TabItem`
  для нужного контрола идёт по **логическому** дереву (`LogicalTreeHelper.GetParent`),
  а не визуальному — иначе вкладку не выбрать и контрол не подсветить.
- **Скрытые цели.** Если контрол не виден (напр. пикер GPU в режиме CPU) — шаг
  показывает карточку по центру без рамки с пометкой «пропусти шаг».

## Проверка
`BrainstormBuddy.exe --coach` → открывает настройки и запускает тур. В логах
(`Coach[i] targetScreen=…`) видно вычисленный прямоугольник подсветки — можно сверить
с UI Automation `BoundingRectangle` реального контрола.
