# План фиксов 2.5.5 — «дичь с RMS», скрытые настройки, окно обучения

Итоги многоагентного расследования (2026-07-13, все выводы подтверждены скептиками-проверяльщиками
по коду; области без проверки перепроверены вручную). Симптомы с реального ноута: бесконечные
красные столбики в шкале АУДИО при паузе динамика, Live RMS «0.000 и не реагирует», ползунок
RMS «ничего не делает», новые настройки авто-калибровки не видны, пузырь обучения живёт отдельно
от настроек.

## Найденные баги

### A. Шкала АУДИО / RMS (4 бага + 3 в авто-калибровке)

| # | Баг | Где | Суть |
|---|-----|-----|------|
| A1 | **Замёрзший RMS** (корень «красной стены») | `AudioBuffer.CurrentRms`, `AudioCaptureEngine` | WASAPI loopback при паузе рендера перестаёт слать буферы; `_cachedRms` обновляется только в `AddSamples`, распада/таймстампа нет → шкала вечно рисует последнее речевое значение. Live RMS при этом из другого источника → «0.000 при красной шкале». |
| A2 | **Ползунок RMS не влияет на шкалу** | `OverlayWindow.OnTimelineTick` | Порог красного столбика захардкожен `0.010`, высота `/0.05`. Ни ползунок, ни авто-порог в отрисовке не участвуют. |
| A3 | **Live RMS слеп к микрофону** | `AudioCaptureEngine.MixTick` | В dual-режиме событие `RmsUpdated` стреляет только от loopback; ветка микрофона события не шлёт вообще. |
| A4 | **Висящая фраза при паузе** | `AudioCaptureEngine.MixTick` | Динамик замолк посреди фразы → VAD не получает кадров тишины → фраза не закрывается и уезжает в STT только при следующем звуке. |
| A5 | **Авто-калибровка не доезжает при старте** | `App.OnStartup`, `Headless/Program.cs` | Конструктор `AudioBuffer` флаг не принимает; прокидка была только по кнопке «Сохранить». После перезапуска поведение не соответствует конфигу. |
| A6 | **Шумовой пол = трекер минимума** | `AudioBuffer.AddSamples` | Скорости 0.05 вниз / 0.0003 вверх (167:1) → пол сползает к минимумам рваного шума, порог ниже среднего шума → фантомная «речь» (симуляция: 18.7% времени в SPEECH). Восстановление порога после тишины ~100 с. Кадры цифровой тишины рушат пол к клампу. |
| A7 | **Режимы VAD 0–2 мертвы** | `SimpleVad.Config.ForMode` | Отличались только абсолютным RMS-порогом, который всегда перекрыт (авто или ползунком). Выбор «0 — Качество» не влияет ни на что; работает только режим 3 (ZCR). |

## План фиксов

- [x] **F1 (A1):** `CurrentRms` «протухает»: таймстамп последнего кадра, > 400 мс без кадров → 0.
- [x] **F2 (A4+A1):** keepalive: при живом loopback-устройстве и пустой очереди N тиков подряд кормить VAD-буфер кадрами тишины (гистерезис ~120 мс от дребезга очереди) — фразы закрываются по паузе, уровень честно падает.
- [x] **F3 (A3):** `RmsUpdated` = max(микрофон, динамик) каждый тик в dual-режиме.
- [x] **F4 (A2):** порог/подсветка шкалы АУДИО = живой эффективный порог (`GetCurrentEffectiveThreshold`) вместо хардкода 0.010.
- [x] **F5 (A5):** при старте `AudioEngine.UpdateConfig(Config.Audio)` (прокидка авто-калибровки и порога в оба буфера); headless — прокидка из конфига. Дефолт поля `_autoCalibrate` — `false` (включается только прокидкой из конфига): тесты и офлайн-харнессы эндпойнтинга сохраняют детерминированный ручной порог.
- [x] **F6 (A6):** робастный шумовой пол: EMA-сглаживание RMS до трекинга (гасит спайки), игнор цифровой тишины (rms < 1e-4), заморозка пола во время речи (речь не калибрует порог под себя), восстановление вверх за секунды (0.005), не за 100 с.
- [x] **F7 (A7):** режимам VAD вернуть смысл в авто-калибровке: режим задаёт множитель запаса над шумовым полом (0 → 4.5 консервативный … 3 → 2.8 чувствительный + ZCR). В ручном режиме ползунок авторитетен (без Math.Min-капа). Подписи в настройках — честные.
- [x] **F8 (B1):** в авто-режимах прятать ТОЛЬКО слайдер порога тишины и пресет аудио; MinSpeech, pre/post-roll, авто-калибровка, RMS-порог, Live RMS — видны всегда. Текст-подсказку исправить.
- [x] **F9 (B2):** удалить дубль-чекбокс CaptureMic.
- [x] **F10 (C1):** убрать `Topmost="True"`; наследовать Topmost целевого окна (owned-окно и так рисуется над владельцем).
- [x] **F11 (C2):** подписка на `IsVisibleChanged` целевого окна: скрылось → пузырь скрыт; отписки в `Finish`.
- [x] **F12 (C3):** `Completed(bool finishedAll)`: тур по оверлею стартует только при полном прохождении; «Пропустить»/Esc → онбординг помечен завершённым без второго тура.
- [x] **F13:** юнит-тест на рваный шум (фантомная «речь» старого пола — репро A6) + прогон всех аудио-тестов.
- [x] **F14:** CHANGELOG 2.5.5, версия установщика, сборка Lite/Full.
